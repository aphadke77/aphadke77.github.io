🦙 Ollama Command Cheat Sheet

Running Models

CommandDescriptionExample
ollama run Run a model interactively in the terminal ollama run llama3.2
ollama run Run with a one-shot prompt (non-interactive) ollama run llama3.2 "What is Docker?"
ollama run Pipe input from stdin cat file.txt | ollama run llama3.2
ollama run Run a specific model version/tag ollama run llama3.2:1b
ollama run Run a multimodal model with an image ollama run llava "describe this" /path/img.jpg
Inside an interactive session, type /bye to exit, /clear to reset context, /help for all commands.

Model Management

CommandDescriptionExample
ollama pull Download a model from the Ollama library ollama pull mistral
ollama pull Pull a specific tag/size variant ollama pull llama3.2:3b
ollama push Push a model to the Ollama library (requires account) ollama push myuser/mymodel
ollama list List all locally downloaded models ollama list
ollama show Show model info (parameters, template, license) ollama show llama3.2
ollama show --modelfile Show the Modelfile for a model ollama show llama3.2 --modelfile
ollama cp Copy a model under a new name ollama cp llama3.2 my-llama
ollama rm Delete a local model ollama rm mistral

Creating Custom Models

CommandDescriptionExample
ollama create Create a model from a Modelfile ollama create mymodel -f ./Modelfile
ollama create Create from a Modelfile in the current directory ollama create mymodel
Modelfile example:
FROM llama3.2
SYSTEM "You are a helpful Python tutor. Keep answers concise."
PARAMETER temperature 0.7
PARAMETER num_ctx 4096
Modelfile InstructionDescription
FROMBase model or GGUF file path
SYSTEMSystem prompt prepended to every conversation
PARAMETERSet model parameters (temperature, top_p, num_ctx, etc.)
TEMPLATEOverride the prompt template
ADAPTERApply a LoRA adapter file
LICENSESpecify the model license
MESSAGEPre-seed conversation history (user/assistant turns)

Server & API

CommandDescriptionExample
ollama serve Start the Ollama server (auto-starts on most installs) ollama serve
ollama ps List currently loaded/running models ollama ps
ollama stop Unload a model from memory ollama stop llama3.2
By default Ollama listens on http://localhost:11434. Set OLLAMA_HOST=0.0.0.0 to expose it on the network.

REST API Quick Reference

EndpointMethodDescription
/api/generate POST Generate a completion (streaming by default)
/api/chat POST Chat with message history
/api/embeddings POST Generate text embeddings
/api/tags GET List local models (same as ollama list)
/api/show POST Show model info
/api/pull POST Pull a model via API
/api/delete DELETE Delete a model via API
/api/copy POST Copy a model via API
/api/ps GET List running models via API
curl example — chat:
curl http://localhost:11434/api/chat -d '{
  "model": "llama3.2",
  "messages": [{"role": "user", "content": "Why is the sky blue?"}],
  "stream": false
}'
Python example (openai-compatible endpoint):
from openai import OpenAI
client = OpenAI(base_url="http://localhost:11434/v1", api_key="ollama")
resp = client.chat.completions.create(
    model="llama3.2",
    messages=[{"role": "user", "content": "Hello!"}]
)
print(resp.choices[0].message.content)

Common Model Parameters

ParameterDefaultDescription
temperature0.8Randomness of output (0 = deterministic, 1+ = creative)
top_p0.9Nucleus sampling — cumulative probability cutoff
top_k40Limit next-token choices to top K
num_ctx2048Context window size in tokens
num_predict128Max tokens to generate (-1 = unlimited)
repeat_penalty1.1Penalize repeated tokens (higher = less repetition)
seed0Random seed for reproducibility
stop—Stop sequences — model stops when it generates these
num_gpuautoNumber of GPU layers to use (0 = CPU only)
num_threadautoNumber of CPU threads

Environment Variables

VariableDescriptionExample
OLLAMA_HOST Address and port the server binds to OLLAMA_HOST=0.0.0.0:11434
OLLAMA_MODELS Directory where models are stored OLLAMA_MODELS=/mnt/models
OLLAMA_ORIGINS Allowed CORS origins (comma-separated) OLLAMA_ORIGINS=*
OLLAMA_KEEP_ALIVE How long models stay loaded after last request OLLAMA_KEEP_ALIVE=10m
OLLAMA_NUM_PARALLEL Max parallel requests handled simultaneously OLLAMA_NUM_PARALLEL=4
OLLAMA_MAX_LOADED_MODELS Max models kept in memory at once OLLAMA_MAX_LOADED_MODELS=2
OLLAMA_FLASH_ATTENTION Enable flash attention (1 = on) OLLAMA_FLASH_ATTENTION=1
CUDA_VISIBLE_DEVICES Select which GPU(s) to use CUDA_VISIBLE_DEVICES=0,1
